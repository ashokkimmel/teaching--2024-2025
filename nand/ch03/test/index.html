3. Test

This is test on chapters 1-3. I have made no effort to keep it to a length that can be done in 50 minutes. Finish what you can.

  1. DMux4Way

    1. Describe the inputs and outputs.
    2. Describe the purpose of the chip and give an example showing how it works.
    3. Implement DMux4Way in HDL. You may use any simpler chip including DMux.

  2. (SubX) The purpose of the HDL chip SubX is to subtract two 16-bit numbers (inputs a and b). You may assume $a>b>0$. (You do not need to think about negatives.) It also determines when the output is even. The outputs are out=a-b and even, which is 1 when the output result is even and 0 otherwise. You may use any of the basic chips including Add16, but not the ALU.

  3. (BigN) The BigN chip takes in a 16-bit number n, and gives the output of 1 when $n \ge 2^{10}$. Write the BigN chip in HDL. You may use any of the chips we have made.

  4. What are the inputs to an ALU? Give their names, bit width, and a brief purpose if necessary.

  5. Your classmate is debugging their ALU and they are trying to compute x+y using flags=101011 with x=0000 0010 1001 0001 and y=1000 0110 0110 1111. Give the answer a correct ALU would produce.

  6. (Zen Chipmaking.) A young student has two teachers, a math teacher and a computer science teacher. The student asks the same question of both: “Are there more positive numbers or negative numbers?” Give the responses of each, along with an explanation of why they respond that way.

  7. The program counter, PC, has the specification below. Write an HDL implementation.

/**
 * A 16-bit counter.
 * if      reset(t): out(t+1) = 0
 * else if load(t):  out(t+1) = in(t)
 * else if inc(t):   out(t+1) = out(t) + 1
 * else              out(t+1) = out(t)
 */
CHIP PC {
    IN in[16], reset, load, inc;
    OUT out[16];
    // ...
    }

Harder

  1. You are trying to design a new kind of tamper-resistant memory. The basis will be a variant of the Register chip that still gives the same answer in the face of a “one bit error” - that is, if a single internal component changes between a 0 and a 1, it should still give the original result.

    There is a hint available for this problem.
